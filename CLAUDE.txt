# Typology Classification Agent (TCA) — Project Context

## Project Overview

The Typology Classification Agent (TCA) is a GCP/Vertex AI-based AI agent being developed
at KeyBank within the Financial Crimes Risk Management team. It is Phase 1 of a two-phase
AML multi-agent SAR automation system. The agent's purpose is to learn to classify AML
typologies from historical SAR narratives with ground truth FinCEN tags, and then apply
that learned Rules Hypothesis to classify unseen SAR narratives.

Phase 2 is the Narrative Drafting Agent, which summarizes Typology Classification Agent
outcomes for AML investigator review.

### ADK Pattern (WORKING)
```python
from dotenv import load_dotenv
load_dotenv('/home/jupyter/custom_agent/typology_agent/.env')

from google.adk.agents.llm_agent import Agent
from google.adk.runners import Runner
from google.adk.sessions import InMemorySessionService
from google.genai.types import Content, Part

async def call_gemini():
    agent = Agent(
        name="tca_agent",
        model="gemini-2.5-flash",
        instruction="system prompt here",
    )
    session_service = InMemorySessionService()
    await session_service.create_session(
        app_name="tca", user_id="user1", session_id="session1"
    )
    runner = Runner(agent=agent, session_service=session_service, app_name="tca")
    async for event in runner.run_async(
        user_id="user1", session_id="session1",
        new_message=Content(parts=[Part(text="prompt here")])
    ):
        if hasattr(event, 'content') and event.content:
            for part in event.content.parts:
                if hasattr(part, 'text') and part.text:
                    print(part.text)
```

## Architecture Overview

The TCA pipeline has three run modes driven from main.py:

### Mode 1: induct
1. Pull historical SAR narratives from BigQuery (sar_narrative_tools.py)
2. Pull FinCEN typology tags from BigQuery (fincen_tag_tools.py)
3. Join both datasets on ITEM_ID — labeled training corpus
4. Generate Reading Plan via Gemini (llm_helpers.py)
5. For each narrative (up to limit), run two sequential Gemini calls:
   - Call 1: extract_narrative_structure()
   - Call 2: synthesize_rules()
6. Output: Rules Hypothesis JSON + Reading Plan JSON

### Mode 2: classify
1. Load Rules Hypothesis (from prior induct)
2. Take unseen SAR narrative as input
3. Run classify_narrative() via Gemini
4. Output: ranked typology candidates with confidence scores
5. Routes to Braintrust (eval) or HITL/SME review (production)

### Mode 3: eval
1. Pull held-out narratives + FinCEN tags
2. Run classify() against each case
3. Log expected vs actual to Braintrust for scoring
4. SME qualitative review follows

## SAR Narrative Structure

Real narratives follow this section structure:
1. **Introduction** — alert trigger, accounts, review period, total suspicious amount
2. **Details of Investigation** — customer profile, account breakdowns,
   credits/debits/counterparties
3. **Suspicious Activity List** — itemized transactions (date, amount, type,
   channel, counterparty, direction)
4. **Conclusion** — summary of findings, total suspicious amount
5. **Contents of File** — supporting documentation list
